5 endpoints

Инструкция для POSTMAN

1) Show All USERS;
2) Create USER;
3) Update USER;
4) GET USER by ID;
5) Delete USER by ID;

1) Show All USERS
   Выберите GET дальше нужно прописать http://localhost:8080/api/users
   Возвращает обьект в JSON формате и показывает всех пользователей которые есть в базе данных.
   И обязательно должно выйти STATUS: 200 OK.

2) Create USER
   Выберите POST дальше нужно прописать http://localhost:8080/api/users
   После заходим в BODY и прописываем данные которые нам нужно добавить, таким образом:
   {
     "info": {
       "name": "Jan",
       "sureName": "Klod",
       "age": 55,
       "gender": "Male",
       "telNumber": "123123",
       "address": "moskovskaya street"
     },
     "role": "USER",
     "userName": "Jan",
     "password": "Jan123",
     "orders": []
   }
   Возвращает обьект в JSON формате и добавляет нового пользователя в базу данных.
   И обязательно должно выйти STATUS: 200 OK.

3) Update USER
     Выберите PUT дальше нужно прописать http://localhost:8080/api/users/здесь нужно указать
     ID данные которого мы хотим изменить.

        После заходим в BODY и прописываем данные которые нам нужно обновить, таким образом:
        {
          "info": {
          "name": "Jan",
          "sureName": "Klod",
          "age": 55,
          "gender": "Male",
          "telNumber": "123123",
          "address": "moskovskaya street"
        }

     Возвращает обьект в JSON формате и обновляет данные существующего пользователя в базе данных.
     И обязательно должно выйти STATUS: 200 OK.

4)  GET USER by Id
    Выберите GET дальше нужно прописать http://localhost:8080/api/users/ выбрать ID пользователя которого хотим найти.
    Возвращает обьект в JSON формате и показывает пользователя который есть в базе данных.
    И обязательно должно выйти 200 OK.

5) DELETE USER by Id
   Выберите DELETE дальше нужно прописать http://localhost:8080/api/users/ выбрать ID пользователя которого нужно удалить.
   Возвращает обьект в JSON формате и показывает удаление пользователя в базе данных.
   И обязательно должно выйти STATUS: 200 OK.
